
1. Интерфейс пользователя:
   - Разработка интуитивно понятного пользовательского интерфейса с элементами для ввода данных о заказах, мониторинга статусов производства и вывода отчетов.
   - Реализация функционала для входа в систему с использованием уникального идентификатора читателя.

2. Функциональность приложения:
   - Создание функций управления информацией о заказах на автомобили с указанием сроков исполнения.
   - Возможность отслеживания статусов выполнения заказов, включая этапы производства.
   - Интеграция с базой данных для хранения информации о клиентах, заказах, автомобилях и статусе производства.

3. Технологии и архитектура:
   - Использование языка программирования C# для создания логики приложения.
   - Использование WPF (Windows Presentation Foundation) для разработки пользовательского интерфейса.
   - Интеграция с SQL Server для работы с базой данных.
   - Реализация модульной структуры приложения для удобного расширения функционала.

4. Тестирование и отладка:
   - Написание модульных тестов для проверки работы отдельных компонентов приложения.
   - Проведение интеграционного тестирования для проверки взаимодействия между компонентами.
   - Отладка для выявления и исправления ошибок.

5. Документация:
   - Написание технической документации о структуре и функциональности приложения.
   - Подготовка руководства пользователя с инструкциями по использованию приложения.
